# 시험암기 제품 원칙

## 최우선 목적: 부분 지식 진단

이 앱의 핵심 차별점은 **한 카드 안에서 사용자가 아는 부분과 모르는 부분을 가림 단위로 구분해 발견하는 것**이다.

일반적인 플래시카드는 카드 전체를 `안다 / 모른다`로 판단한다. 시험암기는 한 카드에 여러 가림을 둘 수 있고, 각 가림을 독립적인 암기 단위로 다룬다. 사용자는 여러 답 중 정확히 어느 부분을 잊었는지 확인하고, 모르는 부분만 다시 학습할 수 있어야 한다.

> 플래시카드는 카드 전체를 평가하고, 시험암기는 카드 안의 기억까지 진단한다.

## 제품 모델

- **카드**는 서로 관련된 내용을 묶는 그릇이다.
- **가림**은 사용자가 실제로 회상하고 판정하는 기본 학습 단위다.
- 카드의 학습 상태와 진행률은 그 안에 있는 가림들의 상태를 요약한 결과다.
- 한 카드에 여러 가림을 만드는 기능은 부가 기능이 아니라 제품의 중심 기능이다.

## 반드시 지켜야 하는 원칙

1. 학습 판정은 카드 전체보다 가림 단위를 우선한다.
2. 사용자는 각 가림을 `알고 있음 / 다시 학습`으로 구분할 수 있어야 한다.
3. 복습은 이미 아는 부분보다 모르는 가림에 집중해야 한다.
4. 진행률과 결과는 가능한 한 카드 수보다 가림 상태를 먼저 설명해야 한다.
5. 카드 생성 과정은 한 카드에 여러 가림을 쉽고 빠르게 지정할 수 있어야 한다.
6. 관련된 답을 불필요하게 여러 플래시카드로 쪼개도록 강요하지 않는다.
7. 새 기능과 시각적 개선은 부분 지식 진단을 더 명확하게 하거나 더 적은 조작으로 가능하게 해야 한다.

## 피해야 하는 방향

- 여러 가림이 있어도 카드 전체에 하나의 `외웠어요` 상태만 저장하는 구조
- 사용자가 어떤 가림을 틀렸는지 잃어버리는 진행 방식
- 틀린 가림 하나 때문에 이미 아는 가림까지 매번 처음부터 반복시키는 방식
- 다중 가림을 숨기고 일반적인 앞면/뒷면 플래시카드를 중심 기능처럼 보이게 하는 UI
- 카드 수만으로 학습 진척을 표현해 부분적으로 아는 상태를 감추는 지표
- 핵심 진단과 무관한 화면, 단계, 장식 또는 선택지를 추가해 학습과 입력을 느리게 만드는 변경

## 모든 수정 전 확인할 질문

다음 질문 중 하나라도 명확히 답할 수 없다면 변경 목적을 다시 검토한다.

1. 이 변경은 사용자가 어느 부분을 알고 모르는지 더 정확히 발견하게 하는가?
2. 여러 가림의 독립적인 상태를 보존하는가?
3. 모르는 가림만 다시 학습하는 흐름에 도움이 되는가?
4. 다중 가림을 만들고 확인하는 조작을 더 단순하게 하는가?
5. 카드 전체를 한 번에 판정하는 일반 플래시카드식 구조로 후퇴시키지 않는가?

## 현재 구현에 대한 태도

현재 구현이 이 원칙을 완전히 충족한다고 가정하지 않는다. 기존 카드 데이터와 사용 흐름을 안전하게 보존하면서, 향후 데이터 구조·학습 판정·진행률·복습 방식이 가림 단위로 수렴하도록 개선한다.
