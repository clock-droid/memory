# Repository instructions

## Product north star

코드, UI, 문구, 데이터 구조 또는 학습 정책을 수정하기 전에 반드시 [`PRODUCT_PRINCIPLES.md`](./PRODUCT_PRINCIPLES.md)를 읽는다.

이 앱의 최우선 목적은 **한 카드 안에서 사용자가 아는 부분과 모르는 부분을 가림 단위로 구분해 발견하고, 모르는 가림만 다시 학습하게 하는 것**이다. 카드 전체를 `안다 / 모른다`로만 처리하는 일반 플래시카드식 구조로 후퇴시키지 않는다.

모든 변경은 다음 중 하나 이상에 기여해야 한다.

- 가림별 지식 상태를 더 정확히 판단한다.
- 모르는 가림만 다시 학습하기 쉽게 한다.
- 여러 가림을 더 빠르고 명확하게 만든다.
- 가림 단위 진행률과 결과를 더 이해하기 쉽게 전달한다.
- 위 목적을 유지하면서 불필요한 조작과 인지 부담을 줄인다.

## 코드 구조

코드를 수정하기 전에 [`ARCHITECTURE.md`](./ARCHITECTURE.md)를 읽는다. `src/`는
`domain → sync → state/actions → views` 한 방향 의존으로 나뉘어 있고, 새 코드는
그 규칙에 맞는 폴더에 넣는다. 특히 가림 단위 로직은 `src/domain/hides.ts`에 모은다.

변경을 완료할 때는 해당 변경이 제품 원칙에 어떻게 부합하는지 검토하고, 핵심 경로를 검증한다.
